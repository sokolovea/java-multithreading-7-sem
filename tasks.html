1. Основы работы с Maven

Цель работы: освоить создание проектов на языке Java и научиться создавть автотесты.

Maven является системой сборки проектов, ширико распространенной в промышленности. Он выполняет несколько функций:

  • автоматическую загрузку и подключение зависимостей (в том числе - транзитивно);

  • сборку проекта для создания результирующего модуля (jar, war и проч.);

  • выполнение дополнительных операций во время сборки.

Рассмотрим простейший файл конфигурации Maven pom.xml:

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <version>0.1-SNAPSHOT</version>
    <artifactId>simple-project</artifactId>
</project>

Запустим сборку проекта

mvn clean install

В результате Maven создаст директорию target, в которой будет расположен результирующий файл сборки - simple-project-0.1-SNAPSHOT.jar.

Исходные коды проекта по умолчанию должны храниться в директории src/main/java.

Как видно, Maven не находи тестов для запуска, потому и не выполняет их. Добавим простейший тест (src/test/java/FirstTest.java):

package com.example.simple.project;

import org.junit.Assert;
import org.junit.Test;

/**
 * Unit test to cover built-in Java operations.
 */
public class FirstTest {

    @Test
    public void testPlusOperator() {
        Assert.assertEquals(5, 2 + 3);
    }
}

Для того, чтобы использовать сторонние библиотеки, их надо добавить в зависимости в pom.xml:

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <version>0.1-SNAPSHOT</version>
    <artifactId>simple-project</artifactId>

    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>

</project>

В зависимости явно добавлено область видимости (scope) - тест. Таким образом, эти библиотеки не будут доступны промышленному коду, но могут быть использованы в процессе сборки для выполнения тестов.

2. Реализация сложной вычислительной задачи

Цель работы: освоение навыков написания сложных алгоритмических задач, а также теоретической и практической оценки их вычислительной сложности.

Эта лабораторная работа посвещана первоначальной реализации сложного вычислительного процессора с целью последующего анализа параллельных алгоритмов решения той же задачи. В результате этой работы, необходимо определить T1 для нескольких параметров точности вычисления, чтобы потом использовать это значение при анализе параллельных алгоритмов.

Задание на лабораторную работу

Необходимо вычислить значения по варианту. При этом необходимо, выполнить временные замеры, чтобы оценить T1. Класс рассчета задачи необходимо сформировать таким образом, чтобы в конструкторе он принимал погрешность вычисления.

Выполнить вычисление согласно заданию многократно, подобрав такие значение погрешности, чтобы вычисления занимали от 1 до 10 секунд.

Задания в этой лабораторной работе также будет выполняться в следуюих лабораторных работах с использованием различного распараллеливания.

Варианты выполнения лабораторной работы:

  1. вычислить определенный интеграл sin(x)*x от 0 до 1

  2. вычислить Pi через площадь круга, используя площать под графиком правого верхнего сектора круга

  3. вычислить Pi через площадь круга, использую площадь вписанного в круг многоугольника с увеличением количества его граней

  4. вычислить Pi по методу Монте-Карло

  5. вычислить сумму бесконечного сдохящегося ряда f(x) = 1/x * sin(x), Найти сумму всех f(x), x принадлежит N

  6. вычислить вероятность того, что броски кубика 20с10 будут больше 80 методом Монте-Карло

  7. вычислить вероятность того, что броски кубика 10с10 со взрывами будет больше 80

  8. реализовать алгоритм быстрой сортировки (QuickSort)

  9. реализовать алгоритм сортировки пузырьком.

3. Создание потоков

Цель работы: освоение навыков создания и запуска потоков.

В данной лабораторной работе предстоит изучить основные классы, используемые для многопоточного программирования в Java.

Целью данной работы является освоение основных классов и конструций, использующихся для создания потоков и выполнения программы в многопоточном режиме.

Для создания нового потока в простейшем случае используется класс Thread. Можно как переопределить его метод run(), так и передать в конструкторе объект, реализующий интерфейс Runnable().

Для справки следует использовать документацию Java: Thread, Runnable.

Задания на лабораторную работу

Исходный алгоритм рассчета необходимо дописать таким образом, чтобы он выводил прогресс своей работы на стандартный вывод (из рассчета, чтобы полное вычисление выдавало 10-20 сообщений о прогрессе в командную строку). Строка прогресса должна содержать долю выполненных на данный момент вычислений (в процентах). Каждое сообщение о прогрессе вывести в отдельной строке.

4. Основной интерфейс работы с потоками

Цель работы: освоение основного интерфейса работы с потоками в Java.

В этой лабораторной работе более доссканально будет изучет жизненный цикл потока и действия, которые возможно с ним собвершать.

Необходимо изучить следующее:

  • публичные вызовы, объявленные в классе Thread,

  • исключение InterruptedException,

  • перехват исключений, вызванных в потоке с помощью UncoughtExceptionHandler,

Теоретическая часть описана в главе Thread Objects руководства по Java.

Задание на лабораторную работу

Создать программу, которая предоставит пользователю интерфейс командной строки в процессе ее выполнения. Таким образом, после запуска программы, пользователь должен увидеть приглашение командной строки. Необходимо реализовать следующие команды:

  1. start <n> - запускает задачу с параметром времени n. Под параметром времени следует понимать величину, влияющую на вычислительную сложность алгоритма (число шагов, требуемая точность и т.р.). Команда должна вывести номер запущенной задачи (локальный идентификатор, используемый только в командной строке).

  2. stop <n> - останавливает задачу с номером n;

  3. await <n> - консоль блокируется до момента завершения задачи с номером n;

  4. exit - завершает выполнения всей программы

Под задачами следует понимать выполнение вычислительно-сложной задачи из предыдущей лабораторной работы согласно своему варианту.

9. Система массового обслуживания

Цель работы: осовение навыков разработки систем массового осбуживания.

Системами массового обслуживания называются системы, удовлетворяющие массовый спрос в форме так называемых заявок, поступающих в случайные моменты времени и образующих входящий поток заявок. Обслуженные заявки образуют выходящий поток обслуженных заявок. Примеры СМО: телефонные станции, ремонтные мастерские, билетные кассы, магазины и т.д.

В рамках лабораторной работы предполагается использование средств синхронизации, освоенных в рамках предыдущих лабораторных работ.

Задания на лабораторную работу

Необходимо реализовать одну их систем массомого осбуживания, согласно варианту задания. После реализации, необходимо реализовать тестирование этой системы массового обслуживания:

  • Протестировать основные и краевые сценарии функционирования системы.

  • Провести стресс-тестирование. В нем заявки в СМО должны отправляться из множества потоков одновременни. Для СМО необходимо произвести проверку результирующего состояния СМО на целостность.

В рамках данной лабораторной работы СМО не должна иметь собственных внутренних потоков. Вся обработка заявок должна происходить в "клиентских" потоках.

Варианты заданий.

  1. Биржа валют. В бирже существует некоторый набор валют и валютных пар, образованных этими валютами. Множество валютных пар может быть меньше декартового произведения множества валют с самим собой. Пользователи отправляют на биржу заявки (ордера) на покупку или продажу валютной пары по определенной цене (продать не дешевле лимита, купить - не дороже). Как только на бирже появляются 2 ордера которые можно выполнить друг напротив друга (возможно, частично), биржа должна заключить сделку. В результате сделки, между пользователями осущетсвляется преевод соответствующих валют. Ордера пользователей уменьшаются на сумму сделки. Необходимо реализовать следующие вызовы в публичном API СМО:

    • создание клиента;

    • ввод-вывод средств клиента;

    • создание заявки на куплю/продажу;

    • запрос открытых на данный момент заявок;

    • запрос состояния клиента (сколько денег в каких валютах у него есть на данный момент).

  2. Проверка крединой способности клиента. Перед выполнением действия по заявки клиента, организация должна выянить, есть ли у этого клиента достаточно средств для оплаты заявки, в случае ее выполнения. У каждого клиента есть определенный счет. Множество пользователей банка обращается к службе кредитных способностей клиента для того, чтобы заблокировать необходимую для сделки сумму денежных средств (СМО может ответить отказом, если денежных средств недостаточно) и освободить денежные средства (сделка отменилась, либо клиент внес дополнительную сумму денег). Разные пользователи могут запрашивать разные операции с одним и тем же клиентом по разным валютам одновременно. Необходимо реализовать следующие вызовы в публичном API СМО:

    • создание клиента;

    • ввод-вывод денежных средств клиента;

    • блокировка/снятие блокировки с дережных средств клиента;

    • запрос состояния клиента (количество дережных средств (доступных и заблокированных).

  3. Онлайн-магазин. Дан магазин. У него есть список товаров и некоторое количество денежных средств на счету. Удаленные пользователи приходят и оставляют заявки на покупку товаров. В результате, если товаров достаточно, то заявка удовлетворается, товар резервируется за пользователем для доставки, а с пользователя списываются денежные средства. Если товаров недостаточно, то пользователь получает собщение об отказе. Множество пользователей одновременно оставляют заявки на покупку разных товаров. Необходимо реализовать следующие вызовы в публичном API СМО:

    • создание клиента;

    • добавление товаров в магазин;

    • покупка товаров клиентом;

    • запрос состояния клиента (сколько денежных средств он потратил);

    • запрос состояния магазина (сколько товаров есть в наличие).

При реализации СМО не требуется реализовывать никакой дисциалины обслуживания.

В лабораторной работе не требуется осуществления общения источников заявок к СМО посредством вычислительных сетей, достаточно реализовать интерфейс между клиентами и СМО в вида API на языке программирования.

В API системы должны быть реализованы вызовы, описывающие текущее состояние ее элементов, по которым можно создать полную картину состояия СМО. Используйте эти выводы при тестировании, чтобы обеспечить целостность данных всей системы целиком (в начальном и конечном состоянии).

Таким образом, внутренее функционирование СМО будет полностью асинхронным: заявка от клиента попадает в очередь, обрабатывается в другом потоке.

Согласно основной идее СМО на основе очередей, структуры данных, изменяющиеся в СМО должны быть представлены в виде несинхронизированных объектов, доступ к которым осуществляется из одного потока (обработчика очереди) в монопольном режиме.

  • См документацию к интерфейсу BlockingQueue.

  • Для реализации асинхронных вызовов удобно использовать класс CompletableFuture.

  • Реализацию API создаваемой системы массового обслуживания (код, который будет класть заявки на обслуживание в очередь) и программный код, обрабатывающий заявки (код, который будет брать заявки на обслуживание из очереди), неоходимо разместить в отдельных классах, таким образом, чтобы у этих классов не было взаимных вызовов. В конструкторе оба класса должны получить очередь - единственный интерфейс между этими частями программы.
